# de-ai-rubric.md · 去 AI 味統一裁判表

> 這是一張把「去 AI 味」拆成可執行判準的**單一裁判表**，合併三類反 AI 資產去重而成：
> - **機械層 prompts**（字數 / 黑名單 / 換序等可量化門檻）
> - **長文病徵檢察官**（後設免責、表格佔比、換序測試、三檢察官面板）
> - **品牌語氣樣本**（Voice DNA + Power Moves + 硬檢查；換成你自己的品牌樣本）
>
> 版本：**v1**。機械層數字來源＝實測校準（非估計），見 §1 各條備註。

---

## §0 使用說明

- 本檔 = 三類反 AI 資產「合併去重」的單一裁判表。三者在「去 AI 味」大方向**互補而非互斥**：機械層補量化門檻（字數/黑名單/換序）、病徵檢察官補長文重病徵（後設免責、表格佔比）與唯一可執行的換序測試、品牌語氣樣本補語氣層與 IG 輪播品牌一致性（喻體宇宙、人稱鎖定）。
- **寫審分離（鐵律）**：產文與判定**不可同一次 pass**。寫的人 ≠ 審的人。這張表是**裁判**用的，不是寫手邊寫邊自評用的。
- **減法優先**：去 AI 味 ≈ 減法。修法一律「先刪後改」。一段刪掉後資訊沒少、讀起來更順 = 它本來就是 AI 贅肉。**但只砍 AI 味，不准砍 SEO 資產**（關鍵字標題、問句 H2、內鏈、CTA、長尾深度）與你的品牌語氣資產（見 §2）。
- **分場景**：機械層（§1）全場景通用；語氣層（§2）的溫度與開場判準**依產出類型分流**（見 §2.4 / §2.5）。
- **怎麼跑**：先用 §1.6 的 python snippet 做半自動粗篩（客觀數字），再由獨立裁判過 §2 / §3（人味判斷）。粗篩 FAIL 直接退回，不用進人味關。

## §1 機械層 checklist（可半自動偵測）

> 這一層是**客觀可量化**的，數字是硬條件。用 §1.6 snippet 一次跑完。

### 1.1 句長節奏
- **句長變異係數 CV = stdev/mean ≥ 0.35**（打破等長；門檻由實測三篇 8+ 分 FB 文校準，CV 落 0.39~0.80，取 0.35 留安全邊際）。
- **超過 20 字強制拆句**（唯一給明確字數門檻的一條）。snippet 會列出所有 >20 字的句子當候選。
- **至少 2 個 10 字內短句當停頓點**。
- **輔助防線：句長落在「平均 ±10 字」內的句子比例 ≤ 40%**（純等長 AI 文此比例逼近 100%，人味文實測 14.7%~32.4%）。

### 1.2 聯合黑名單（三類詞彙聯集，逐項標來源，全文 0 命中才過）

| 類別 | 詞條 | 來源 |
|---|---|---|
| A · 過渡/總結詞（12 項） | 首先／其次／此外／總結來說／總的來說／值得注意的是／重要的是／綜上所述／隨著科技發展／相關數據顯示／在這個時代／`不僅僅?是.{0,15}更是`（正則） | 機械層過渡詞 prompt |
| B · 商業黑話/空詞（9 項） | 護城河／本質／賦能／底層邏輯／`真正的[一-龥]`（正則）／核心競爭力／降維／沉澱／未來已來 | 品牌語氣樣本 hardCheck（＝ knowledge_carousel_rubric 硬 checklist 同一份，合併只留一份） |
| C · 後設免責/口徑碎念訊號詞 | 以官方為準／各家說法不一／有來源佐證／我不取一／並列不取一／僅供參考 | 病徵檢察官病徵 2（**最重病徵**，命中可直接 ≥7）；健康/法規必要免責 ≤1 句可留 |

> 說明：A 與 B 詞彙**完全不重疊**（一個是過渡詞、一個是商業黑話），故直接聯集。C 是長文/SEO 場景最常出現，不可省。

### 1.3 對偶/破折號數量門檻
- **破折號反轉句／對偶句型（「你以為 X — 其實是 Y」「不是 X 而是 Y」「與其 X 不如 Y」）全篇（含標題/封面）≤ 2 處，超過一律砍到剩 ≤2。**
- 這條化解了衝突：病徵檢察官判「破折號反轉句成癮」為死刑指紋，品牌語氣樣本卻把「你以為 X — 其實是 Y」列為封面**必要動作**。裁決＝**保留 1 處當品牌語氣資產、上限 2 處**。

### 1.4 結構檢測
- **換序測試**（唯一可操作測試法）：隨機抽兩段對調 → 順讀無感 = 命中（FAIL）；讀起來變怪 = 通過（順序鎖死）。
- **表格/清單佔比上限**：敘事型 ≤30%／知識型 ≤45%／硬上限 <50%。
- **平行清單缺承轉偵測**：相鄰段落間無承轉句、像並排清單 = 命中。

### 1.5 標點格式
- 走**逗號流**：日常用「，、」，句子靠換行斷，少用「。？：；…」。
- **禁戲劇冒號開場**（「關於 X：」「真相：」）→ 改逗號流或換行。
- **禁英文半形標點**；句尾可偶用「～」軟化。

### 1.6 可直接跑的檢測 snippet（句長 CV ＋ 黑名單 grep）

> 用法：`python3 de_ai_check.py <文章.md>`。
> 正文抽取預設用 FB 文的「從這裡開始複製到 FB … 留言區第一則」標記；沒有標記就吃全文。逗號流不切句（逗號是品牌語氣特徵）。

```python
#!/usr/bin/env python3
# de_ai_check.py — 去 AI 味機械層半自動粗篩（句長 CV + 聯合黑名單）
import re, sys, statistics

# ---- §1.2 聯合黑名單（逐類標來源）----
BL_TRANSITION = ["首先","其次","此外","總結來說","總的來說","值得注意的是",
                 "重要的是","綜上所述","隨著科技發展","相關數據顯示","在這個時代"]  # 過渡詞
BL_TRANSITION_RE = re.compile(r"不僅僅?是.{0,15}更是")                              # 第12項(正則)
BL_BUZZWORD  = ["護城河","本質","賦能","底層邏輯","核心競爭力","降維","沉澱","未來已來"] # 空詞
BL_TRUEX_RE  = re.compile(r"真正的[一-龥]")                                         # 「真正的X」
BL_HEDGE     = ["以官方為準","各家說法不一","有來源佐證","我不取一","並列不取一","僅供參考"] # 病徵2

def load_body(path):
    text = open(path, encoding="utf-8").read()
    m = re.search(r'從這裡開始複製到 FB(.*?)留言區第一則', text, re.S)
    body = m.group(1) if m else text
    lines = [l.strip() for l in body.split('\n')]
    drop = ('🙏 致謝','PS：','---','http')
    lines = [l for l in lines if l and l != '■' and not l.startswith('##')
             and not any(l.startswith(p) for p in drop)]
    return lines

def sentences_of(lines):
    para = '\n'.join(lines)
    raw = re.split(r'(?<=[。！？])', para)          # 只用句末標點切句，逗號不切
    return [s.strip() for s in raw if len(s.strip()) > 1]

def check(path):
    lines = load_body(path)
    full  = '\n'.join(lines)
    sents = sentences_of(lines)
    lens  = [len(s) for s in sents]

    # --- 句長節奏 (§1.1) ---
    mean = statistics.mean(lens); cv = statistics.pstdev(lens)/mean
    within10 = sum(1 for L in lens if abs(L-mean) <= 10)/len(lens)
    over20   = [s for s in sents if len(s) > 20]
    short10  = [s for s in sents if len(s) <= 10]

    # --- 聯合黑名單 (§1.2) ---
    hits_t = [w for w in BL_TRANSITION if w in full] + BL_TRANSITION_RE.findall(full)
    hits_b = [w for w in BL_BUZZWORD  if w in full] + BL_TRUEX_RE.findall(full)
    hits_h = [w for w in BL_HEDGE     if w in full]

    # --- 結尾昇華：末 2 句加驗「過渡詞∪空詞」黑名單 (§2.5) ---
    tail = '\n'.join(lines[-2:])
    tail_hits = ([w for w in (BL_TRANSITION+BL_BUZZWORD) if w in tail]
                 + BL_TRANSITION_RE.findall(tail) + BL_TRUEX_RE.findall(tail))

    print(f"== {path.split('/')[-1]} ==")
    print(f"句數 {len(sents)} | 平均 {mean:.1f} 字 | 最短/最長 {min(lens)}/{max(lens)}")
    print(f"[§1.1] CV={cv:.3f}  (門檻 ≥0.35 → {'PASS' if cv>=0.35 else 'FAIL'})")
    print(f"[§1.1] within±10 = {within10:.1%}  (門檻 ≤40% → {'PASS' if within10<=0.40 else 'FAIL'})")
    print(f"[§1.1] >20字待拆句 {len(over20)} 句 | ≤10字停頓句 {len(short10)} 句"
          f"  (停頓 ≥2 → {'PASS' if len(short10)>=2 else 'FAIL'})")
    print(f"[§1.2-A 過渡詞]   {hits_t or 'PASS(0)'}")
    print(f"[§1.2-B 空詞]     {hits_b or 'PASS(0)'}")
    print(f"[§1.2-C 後設免責] {hits_h or 'PASS(0)'}")
    print(f"[§2.5 結尾昇華]   {tail_hits or 'PASS(0)'}   結尾片段: {tail[:80]!r}")

if __name__ == "__main__":
    for p in sys.argv[1:]:
        check(p); print()
```

> 這只是**粗篩**：機械層全 PASS ≠ 沒 AI 味（換序測試、破折號密度、溫度分流仍要人工過 §2/§3）。粗篩任一 FAIL 直接退回。

## §2 語氣層（吃樣本，人工判準為主）

### 2.1 語調複製的依存關係
執行**語調複製**時，樣本源預設 =「本檔 §2.2 的品牌語氣樣本 + 你的歷史 FB/Threads 貼文」。「語調複製」是方法，語氣樣本是這方法要貼的素材，不是兩條互斥規則。備註：樣本選「最像你的文字」，不是「最好的文字」。

### 2.2 品牌語氣 DNA（示範樣本；換成你自己的）
> 下面是一組「白話文翻譯官／說書人」風格的**示範**語氣樣本。**用時換成你自己品牌的樣本**——這裡只示範「一份 Voice DNA 長什麼樣、要記哪些維度」。

把硬內容嚼碎餵給**特定外行族群**，招牌動作「翻成人話就是…」。冷靜、誠實、不 hype，靠「有用＋看得懂」驅動收藏分享。
- **人稱**：第一人稱在場（「我逐行讀完」）帶讀者走思考；第二人稱鎖族群（「給用過 AI、但沒寫過 code 的你」）逼對號入座；「我們」只給社群/團隊。
- **真實感**：具體＞抽象，用真數字、真出處、真動作（弄、搞、debug、跑、改）；自嘲＞宣言。誠實人設**前置**（先承認「這不是我的研究，我只是整理者」），用自降姿態換信任。
- **Power Moves 速記**：反差認知鉤／負向賭注／唯一鐵律＋大部分人搞反／翻成人話轉場／生猛生活比喻／權威反證當懸念／短句破折號急煞／兩個詞口訣收尾／誠實前置＋反向勸退／關鍵字 CTA。

### 2.3 喻體紀律
所有比喻共用**同一組喻體宇宙**（示範：師傅＝生成、獨立試吃員＝驗收、量尺＝硬訊號、學生改自己考卷＝reward hacking），全套不換喻體，記憶不被稀釋；抽象先具象（「省 90%」→「10 杯咖啡付 1 杯」）。**換喻體（師傅講一半跳去用齒輪/引擎/生態系）= 重寫。**

### 2.4 溫度分流規則
> 化解「品牌語氣要冷靜克制」vs「病徵檢察官把砍光溫度判為平板腔」的表面矛盾——兩者分屬不同產出場景，**依產出類型分流，兩條不互相覆蓋**。

- **分支 A — 翻譯官人設（ELI5 / IG 輪播 / 簡報文字）**：冷靜克制，**禁 over-narrate 情緒**（「那一刻我突然…」「那種感覺真的會讓你…」）。人設是白話文翻譯官，不灑狗血、不 hype。
- **分支 B — SEO / 長文敘事**：**保留情感與畫面、禁平板腔**。溫度＝人味；為減法把溫度砍光、讀起來乾淨卻像中性說明書＝另一種 AI 味，照樣扣分。教學預告/signposting、鋪陳反問句、成語驚嘆號、實用提醒段在分支 B 一律**加分不扣分**。

### 2.5 開場鋪陳邊界（具體性判準）
> 化解「禁 staged 開場」vs「護教學預告」——用**是否具體**當判準，不是二選一。

- **可留**：有**具體路徑/內容清單 + 熱情語氣**的開場（例：「這篇帶你溫度→杯型→…一次上手！」）——有資訊增量的教學預告是熱情老師開場。
- **要砍**：**純吊胃口、無資訊增量**的軟開場（「說個我以前看不懂的事」「你有沒有發現」「分享一個觀察」）→ 改成直接給反差結論。
- **結尾昇華檢測**：結尾停在事實/具體技能上；**末 2 句**額外套 §1.2-A∪B 黑名單（見 §1.6 snippet），因為空詞正是「昇華式」結尾常用字。空泛升格收尾（「永遠值錢」「未來已來」）除非綁具體技能/可查證動作否則砍。

### 2.6 品牌人格由故事證明、非宣稱
品牌人格要被**故事/觀點證明**，不能只被「宣稱」。搭配人稱鎖定 + 誠實前置 + 自嘲＞宣言。此項兩層互補（一層談「證明 vs 宣稱」、一層談「人稱/自嘲」），都留、不去重。

## §3 P7 自檢閘（出貨前，寫審分離，獨立 pass）

> 這一關由**獨立裁判**（另一個 agent / 另一次 pass，預設有罪推定）跑，不由產文的同一次 pass 自評。

1. **主體＝長文病徵檢察官完整流程**（若你有裝對應的長文去 AI 味 skill，如 article-joker，直接引用它；沒有就照下面判準手動跑）：八病徵評分（每項 0-10、附原文 span）＋三檢察官面板（結構/語感/人味）＋死刑判決 line。發布門檻＝**代表總分 ≤3 ＋ 八病徵全綠 ＋ 換序測試不通過（順序鎖死）**。
2. **併入機械層 P7 判準**：找「過於圓滑無立場句／日常不會說出口的詞／太對稱像範本的段落」，逐一改寫，**輸出雙版本（標修改處版 + 乾淨版）**。
3. **併入品牌語氣硬檢查逐條二元判定**（僅套用於 IG 輪播/社群產出；每段 ≥8 且總分 ≥8.5 才放行）。
4. **反群體迷思條款**：三檢一致 ≠ 客觀無 AI 味，可能只是模型共同偏見；特別點名兩種偽勝利——「脫了規格書皮沒脫骨」「逃進反 AI 範文腔」。需要更獨立第二意見時把稿子丟另一個模型跨模型重批。

## §5 一句話總結

**去 AI 味 = 減法 + 寫審分離。** 先用 §1.6 snippet 砍掉客觀可測的機械味（等長句、過渡詞、空詞、後設免責），再由獨立裁判過 §2 語氣分流與 §3 病徵自檢；粗篩任一 FAIL 直接退回，不進人味關。
