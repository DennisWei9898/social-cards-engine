# 知識型 IG 圖卡 · 通用撰寫 Rubric（任何品牌 / 任何主題）

> 來源：2026-07 某食品品牌（範例）「堅果完全指南」實戰（10 張，經多鏡頭社群診斷從 6.5 → 改版）萃取。
> 適用：把任何 SEO 文章 / 教學 / 主題，轉成「會被存、被傳、被買」的知識型 IG 輪播。不限品牌、不限題材。
> 搭配：`voice_rubric.md`（管語氣 / 人味）並列——**這份管「結構與社群機制」**。
> 演算法量尺：`~/.claude/skills/social-post/references/evaluation.md`（2026 訊號權重）。

## 🔧 v1 研究修正（2026-07-01，跨平台 workflow 驗證後，此段最高優先）
- **策略重心＝sends 與 saves 並重、但 sends 先**：IG 官方（Mosseri）自證「**sends per reach 私訊分享率**」是最強分發訊號，saves 排第二。封面/CTA 要**同時**做「轉傳鉤（提醒誰）＋收藏誘因」。
- **尺寸＝知識型用直式 4:5（1080×1350）**滿版（IG/Threads/小紅書全偏好）；只有 你個人分享文維持 1:1 方形。
- **Hashtag ≤5 個**（精準分類，非塞熱門；超過易被判 spam/擋發文）。
- **字級單位是 px 非 pt**：body≥24px、headline≥36–40px；每張 10–15 字更佳，30 是上限。
- **刪除杜撰數字**：封面80%權重 / >12掉40%完讀 / 第一小時80% / saves=讚3倍 / 完讀>60% 都是部落格迷思，**只當方向參考、不寫死、文案不拿來當承諾**。
- 連結導 bio/限動貼紙（一般帳號正文本就不能放可點連結，非「連結扣觸及」）。

## 🔩 演算法訊號優先序（設計時的北極星）

實證權重（IG 官方定性）：**私訊分享 sends > 儲存 saves > 5 字長留言 > 公開分享 > 一般留言 > 讚≈0**。
知識型輪播的設計目標：**① 讓人「傳給某個特定的人」（sends，最高）② 讓人「存起來」（saves）③ 引導留言關鍵字**。讚不是目標。

## 🧱 結構模板（8–10 張，教育型甜蜜點）

| 位置 | 角色 | 要點 |
|---|---|---|
| 1 封面 | 鉤子＋署名＋收藏訊號 | 損失規避/好奇缺口標題（非類目名）＋品牌名＋信任徽章＋「📌建議收藏」＋SWIPE |
| 2 | 認知地基 | 「這是什麼 / 為什麼該在意」，比喻先行 |
| 3 | 數據/主張卡 | 把數字升級成觀點（如「熱量差不多，關鍵在取代」）|
| 中段 | **照片喘息卡** | 每 2–3 張文字後插一張真實照片卡打斷節奏 |
| 工具卡群 | 份量/挑選/保存/對比 | 「之後會回來查」的工具性內容＝save 引擎 |
| 倒數第 2 | **速查總表卡** | 把分散資訊併成「一張截圖帶走」，標「存這張」 |
| 分享卡 | 點名對象 | 「家裡有X？把這張傳給他 →」觸發最高權重的私訊分享 |
| 最後 | 品牌 CTA | USP 主張＋信任證據＋一主一輔 CTA＋「連結在主頁」 |

## ✅ 10 條硬 checklist（發布前逐條過）

1. **封面是「鉤子」不是「類目」**：命中好奇/FOMO/防禦/損失規避之一；5–8 字大標最大最先讀到；避免「X 完整解析」這種目錄式。
2. **封面即帶品牌署名＋信任徽章**：單張被分享時仍認得出是誰（logo/獎章/品牌名，非浮水印級）。
3. **封面明示「可存工具」＋收藏指令**：saves 是讚的 3×，別只在結尾才喊收藏。
4. **每張一個概念**：主標一句、內文 <30 字、主數字放大、note 降級為小字。
5. **9 張有節奏起伏**：照片／文字／數據交錯，中段不要連 4 張以上同版型。
6. **主副層級對比夠**：副標/內文在底色上要達可讀門檻（暖底忌太淺灰，往深靠），手機縮圖不糊。
7. **至少一張「速查/總表卡」**：讓人一張截圖帶走、利於單張轉傳。
8. **結尾 CTA 一主一輔、動詞前置、附收益理由**：可 +20–30% 互動；別三條並列。
9. **針對「該提醒誰」的卡給轉傳指令**：點名對象（長輩/新手/同事）拉私訊分享——最高權重訊號。
10. **知識與商品有橋樑**：中段用知識「反證」產品選擇（「這也是我們堅持X的原因」），別到最後才硬接導購。

## 🎨 品牌自適應設計（花叔 Design 紀律）

- **先建 `brand-spec.md`**：從品牌真實素材抽 logo / 產品照 / 色票 / 字型（見 huashu-design 核心資產協議）。**不套 你個人的 navy/espresso 配色**——每個品牌一套。
- **一定放真實照片**：產品照 / 情境照 / 素材照，`file_upload` 或 `<img>` 嵌入；**禁 CSS 剪影假圖**。3–4 張真圖分散在封面、照片喘息卡、品牌卡即可（宁缺毋滥）。
- **一個品牌一套模板 + 一個簽名母題**（如某食品品牌（範例）的金鑽石◇、暖米白+金+明體）。
- 版型參考：`templates/render_cards_warm_eli5_example.py`（你個人）／`example-brand-carousel/render_tyn.py`（品牌精品範本）。

## 🔁 產出流程（5 階段）

1. **抽素材**：讀來源文章 → 抽可上卡的內容＋數據；curl 品牌官網下載真實照片 → 寫 `brand-spec.md`。
2. **拆卡**：套上面結構模板，8–10 張。
3. **產圖**：HTML+CSS 模板 → Chrome headless 1080×1080 PNG（真實照片嵌入）。
4. **社群診斷閘（寫審分離）**：對照本 rubric 10 條 + `evaluation.md` 訊號權重打分，指到卡給改法（可用多鏡頭 workflow：鉤子/收藏/可讀/品牌 + 研究）。不過就改。
5. **文案 + 發布**：`social_captions.md`（首句呼應封面鉤子、連結導留言/bio）；發文後看 **收藏數 / 私訊分享 / 連結點擊率>1% / 留言關鍵字人數**（別看讚）。

## 一句話
知識型圖卡的贏法＝**封面下對鉤子、中段給工具、結尾給總表與轉傳指令、全程掛品牌**，對著演算法最貴的「私訊分享 + 收藏」去設計。
